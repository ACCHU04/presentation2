Quick Definition

Greedy Algorithm — an approach that makes the locally optimal (best) choice at each step with the hope it leads to a global optimum. Suitable for problems with greedy-choice property and optimal substructure.

Uses of Greedy Algorithms

  • Used in optimization problems where decisions must be made step-by-step.
  • Ideal for scheduling problems (task selection, activity selection).
  • Used in network algorithms like Prim’s and Kruskal’s (Minimum Spanning Tree).
  • Applied in Huffman Coding for data compression.
  • Useful in resource allocation problems.
  • Coin change and currency-based problems (if system is canonical).

Advantages of Greedy Algorithms

  • Simple and easy to implement.
  • Very fast — usually O(n log n) or O(n).
  • Requires less memory compared to dynamic programming.
  • Provides optimal solutions for many real-world problems.
  • Efficient for large input sizes.
  • Step-by-step decision-making makes debugging easier.

General Greedy Flowchart

Example 1 — Coin Change (Indian coins)

Greedy strategy: pick the largest coin possible at each step.

₹5
₹2

Pseudocode (simple):

coins = [10, 5, 2, 1]
function minCoins(amount):
  count = 0
  for coin in coins:
    while amount >= coin:
      amount -= coin
      count += 1
  return count

// Example: amount = 28 -> 10,10,5,2,1 = 5 coins

Example 2 — Activity Selection

Sort activities by finish time; pick earliest-finishing activity that starts after the last picked activity's finish.

Pseudocode:

function activitySelection(start[], finish[], n):
  sort activities by finish time
  select first activity
  last_finish = finish[0]
  for i from 1 to n-1:
    if start[i] >= last_finish:
      select activity i
      last_finish = finish[i]

Example 3 — Fractional Knapsack

Pick items in descending order of value/weight ratio. If the next item doesn't fully fit, take the fraction that fits.

Pseudocode:

function fractionalKnapsack(W, items):
  sort items by value/weight descending
  totalValue = 0
  for item in items:
    if item.weight <= W:
      W -= item.weight
      totalValue += item.value
    else:
      fraction = W / item.weight
      totalValue += item.value * fraction
      break
  return totalValue

Ready-to-use Code (Python)

Paste into any Python file and run.

# Coin change (greedy) - assumes canonical coin system
coins = [10,5,2,1]
def min_coins(amount):
    count = 0
    for c in coins:
        count += amount // c
        amount = amount % c
    return count

# Activity selection
def activity_selection(activities):
    # activities = list of (start, finish)
    activities.sort(key=lambda x: x[1])
    res = [activities[0]]
    last_finish = activities[0][1]
    for s,f in activities[1:]:
        if s >= last_finish:
            res.append((s,f)); last_finish = f
    return res

# Fractional knapsack
def fractional_knapsack(cap, items):
    # items = list of (value, weight)
    items = sorted(items, key=lambda x: x[0]/x[1], reverse=True)
    val = 0.0
    for v,w in items:
        if cap >= w:
            cap -= w; val += v
        else:
            val += v * (cap / w)
            break
    return val

Notes & Tips (Exam)

  • Greedy works only when local choices lead to a global optimum — otherwise counterexamples occur.
  • To prove greedy correctness, use an exchange argument or show optimal substructure.
  • For coin change, greedy fails on some coin systems (e.g., {1,3,4} for amount 6).
